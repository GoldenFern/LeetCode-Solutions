# LeetCode 3: 无重复字符的最长子串 (Longest Substring Without Repeating Characters)

## 1. 题目描述

给定一个字符串 `s` ，请你找出其中不含有重复字符的 **最长子串** 的长度。

**示例 1:**

> 输入: s = "abcabcbb"
> 输出: 3 
> 解释: 因为无重复字符的最长子串是 "abc"，所以其长度为 3。

**示例 2:**

> 输入: s = "pwwkew"
> 输出: 3
> 解释: 因为无重复字符的最长子串是 "wke"，所以其长度为 3。请注意，你的答案必须是 **子串** 的长度，"pwke" 是一个子序列，不是子串。

**提示：**

* $0 \le s.length \le 5 \times 10^4$
* `s` 由英文字母、数字、符号和空格组成

---

## 2. 算法基本原理：滑动窗口 (Sliding Window)

滑动窗口是处理**连续子串/子数组**问题的经典优化策略。

* **核心思想**：维护一个窗口 $[left, right]$，通过移动 $right$ 边界扩大搜索范围，当满足特定条件（出现重复字符）时，移动 $left$ 边界缩小窗口。
* **时间复杂度优化**：使用哈希表（Dictionary）记录每个字符最近一次出现的位置，从而让 $left$ 指针在遇到重复字符时实现“跳跃式”移动，无需一步步递增，将时间复杂度从 $O(n^2)$ 降低至 $O(n)$。

---

## 3. 数学归纳法对算法的证明

**命题 $P(n)$**：设 $f(i)$ 为以字符串索引 $i$ 结尾的无重复最长子串的长度。则算法在第 $i$ 次迭代后能正确计算出全局最大长度 $\max \{f(0), f(1), \dots, f(i)\}$。

* **基础步骤 (Base Case)**：
  当 $i=0$ 时（字符串只有一个字符），$right=0, left=0$，当前窗口长度为 1。显然，长度为 1 的字符串最长无重复子串就是它本身，命题成立。

* **归纳假设 (Inductive Step)**：
  假设当 $right=k$ 时，算法已经正确找到了以 $k$ 结尾的最长子串，且 $left$ 位于满足无重复条件的最小起始位置。
  当 $right=k+1$ 时，新加入字符 $s[k+1]$：
  
  1. 若 $s[k+1]$ 未曾在当前窗口 $[left, k]$ 出现，则以 $k+1$ 结尾的最长子串长度自然扩大为 $(k+1) - left + 1$。
  2. 若 $s[k+1]$ 在索引 $j$ 处出现过，且 $j \ge left$（即重复字符在当前窗口内），则为了消除重复，新的左边界必须跳至 $j+1$。
  3. 若 $j < left$，说明该重复字符已在当前窗口之外，不会破坏当前窗口的无重复性，无需处理。代码中的 `max(left, char_map[char] + 1)` 完美覆盖了情况 2 和 3。

* **结论**：
  根据数学归纳法，算法在每次迭代中都正确计算了以当前字符结尾的最长合法子串长度，并通过 `max_length` 记录历史最大值，因此对任意长度的字符串均有效。

## 4. 编程中需要注意的小细节 (易错点)

1. **左边界不能回退 (`left` 的更新逻辑)**：
   这是本题最容易踩坑的地方。更新左边界时必须使用 `left = max(left, char_map[char] + 1)`。
   
   * **反例分析**：假设测试用例为 `s = "abba"`。
     * 当遍历到第二个 `'b'` (索引 2) 时，`left` 更新为索引 2。
     * 当遍历到第二个 `'a'` (索引 3) 时，`'a'` 之前在索引 0 出现过。如果直接写 `left = char_map['a'] + 1`，`left` 会变回 1。这导致窗口从向右滑动变成了向左回退，从而错误地把两个 `'b'` 包含在了窗口内。使用 `max` 函数可以强制 `left` 只能向右单调移动。

2. **哈希表的灵活性**：
   在 Python 中，使用字典 `{}` 作为哈希表非常方便，可以直接通过 `if char in char_map:` 来判断字符是否出现过。每次遍历结束前，都必须执行 `char_map[char] = right` 来刷新字符的最新位置。

3. **空字符串的鲁棒性**：
   初始化 `max_length = 0`。如果输入 `s = ""`，`for` 循环根本不会执行，直接返回 0，逻辑天然闭环，不需要额外的 `if s == "": return 0` 特判。
