# LeetCode 283 & 15: 移动零 & 三数之和

## 1. 题目描述

### 283. 移动零 (Move Zeroes)

给定一个数组 `nums`，编写一个函数将所有 `0` 移动到数组的末尾，同时保持非零元素的相对顺序。
**请注意**，必须在不复制数组的情况下原地对数组进行操作。

> **示例 1:**
> 输入: nums = [0,1,0,3,12]
> 输出: [1,3,12,0,0]

### 15. 三数之和 (3Sum)

给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k` ，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。请你返回所有和为 `0` 且不重复的三元组。
**注意**：答案中不可以包含重复的三元组。

> **示例 1:**
> 输入：nums = [-1,0,1,2,-1,-4]
> 输出：[[-1,-1,2],[-1,0,1]]

---

## 2. 算法基本原理

### 283. 移动零：快慢双指针 (简述)

* **核心思想**：维护两个指针，`fast` 用于遍历数组寻找非零元素，`slow` 用于记录下一个非零元素应该存放的位置。
* **操作**：当 `fast` 遇到非零元素时，将其与 `slow` 指向的元素交换，然后 `slow` 向前移动。这样可以保证 `slow` 左侧的所有元素都是非零元素，且保持了原有的相对顺序。时间复杂度 $O(n)$，空间复杂度 $O(1)$。

### 15. 三数之和：排序 + 对撞双指针 (详述)

* **核心思想**：将 $O(n^3)$ 的暴力枚举优化为 $O(n^2)$。首先对数组进行**排序**，然后遍历数组，固定一个数 `nums[i]`，将问题转化为在剩余的有序数组中寻找**两数之和**等于 `-nums[i]`。
* **对撞指针**：在 `i` 之后的区间 `[i+1, n-1]` 中，设置左指针 `left` 和右指针 `right`。根据三数之和与 $0$ 的大小关系，动态向中间收缩指针。

## 3. 算法证明：为什么双指针不会漏解？

在 15 题中，我们在有序区间内使用对撞指针寻找两数之和。我们可以用 **反证法（缩减搜索空间法）** 来严谨证明该策略的正确性。

**命题**：在有序数组中寻找两数之和等于 `target`，`left` 和 `right` 指针向中间移动的策略绝对不会漏掉任何一个合法解。

**证明**：
假设存在一个合法解 `(L*, R*)` 满足 `nums[L*] + nums[R*] == target`，但被我们的算法漏掉了。
算法初始状态为 `left = 0, right = n-1`。在整个过程中，`left` 单调递增，`right` 单调递减。
如果要漏掉 `(L*, R*)`，必然是在某一步，其中一个指针先到达了目标位置，而另一个指针在到达目标位置前，先一步越过了目标位置。

假设 `left` 先到达了 `L*`，此时 `right > R*`。
此时我们计算两数之和：`sum = nums[L*] + nums[right]`。
因为数组是升序排列的，且 `right > R*`，所以必然有 `nums[right] >= nums[R*]`。

1. 若 `nums[right] == nums[R*]`，则 `sum == target`，算法会记录该解，矛盾。
2. 若 `nums[right] > nums[R*]`，则 `sum > target`。根据算法逻辑，当和大于目标值时，`right` 指针会左移（`right -= 1`）。
   因此，`right` 指针会不断左移，直到 `right == R*`，此时 `sum == target`，算法成功找到该解，矛盾。

同理，若 `right` 先到达 `R*`，`left` 也会不断右移直到 `left == L*`。
**结论**：双指针每次移动都安全地排除了不可能的区间，绝对不会错过任何正确的组合。

## 4. 编程中需要注意的小细节 (易错点)

15 题的难点不在于找到解，而在于**如何高效地去重和剪枝**。

1. **排序是基础**：
   必须先执行 `nums.sort()`。排序不仅是双指针移动的前提，更是后续去重逻辑的基础（相同的数字会挨在一起）。

2. **极致剪枝 (Pruning)**：
   在遍历固定第一个数 `nums[i]` 时，如果 `nums[i] > 0`，因为数组已经排序，后面的数必然都大于 0，三数之和绝对不可能等于 0。此时可以直接 `break` 结束整个循环。

3. **外层循环去重 (固定元素的去重)**：
   当 `nums[i] == nums[i-1]` 时，说明当前固定的数字和上一次一样，这会产生重复的三元组，必须 `continue` 跳过。
   
   * **易错点**：判断条件必须是 `i > 0 and nums[i] == nums[i-1]`，**绝对不能**写成 `nums[i] == nums[i+1]`。如果写成后者，会把 `[-1, -1, 2]` 这种合法的包含重复数字的解给错误地跳过。我们要做的是“不让同一个数字在第一个位置被固定两次”，而不是“不允许三元组里有相同的数字”。

4. **内层循环去重 (双指针的去重)**：
   当找到一个和为 0 的解后，不能仅仅把 `left` 和 `right` 移动一步，必须用 `while` 循环跳过所有连续相同的数字。
   
   * `while left < right and nums[left] == nums[left + 1]: left += 1`
   * `while left < right and nums[right] == nums[right - 1]: right -= 1`
   * **注意**：去重后，指针还需要再各自移动一步（`left += 1`, `right -= 1`），才能真正指向新的不同元素。
