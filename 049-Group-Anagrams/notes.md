# LeetCode 49: 字母异位词分组 (Group Anagrams)

## 1. 题目描述

给你一个字符串数组，请你将 **字母异位词** 组合在一起。可以按任意顺序返回结果列表。

**字母异位词** 是由重新排列源单词的所有字母得到的一个新单词。

**示例 1:**

> 输入: strs = ["eat", "tea", "tan", "ate", "nat", "bat"]
> 输出: [["bat"],["nat","tan"],["ate","eat","tea"]]

**示例 2:**

> 输入: strs = [""]
> 输出: [[""]]

**示例 3:**

> 输入: strs = ["a"]
> 输出: [["a"]]

**提示：**

* $1 \le strs.length \le 10^4$
* $0 \le strs[i].length \le 100$
* `strs[i]` 仅包含小写字母

---

## 2. 算法基本原理：哈希表 + 排序 (Sorting with Hash Map)

本题的核心在于如何找到一个**统一的标识（Key）**，使得所有互为字母异位词的字符串都能映射到同一个 Key 上。

* **核心思想**：两个字符串互为字母异位词，当且仅当它们排序后的字符串相等。
* **算法流程**：
  1. 遍历字符串数组中的每个字符串。
  2. 对当前字符串进行排序，将排序后的结果作为哈希表（字典）的 `key`。
  3. 将原始字符串存入该 `key` 对应的列表中。
  4. 最后返回哈希表中所有 `values` 组成的列表。
* **复杂度分析**：
  * **时间复杂度**：$O(N \cdot K \log K)$，其中 $N$ 是字符串数组的长度，$K$ 是字符串的最大长度。遍历需要 $O(N)$，每个字符串排序需要 $O(K \log K)$。
  * **空间复杂度**：$O(N \cdot K)$，用于存储哈希表中的所有字符串。

---

## 3. 算法证明：基于等价关系的规范化表示

**证明目标**：证明“排序后的字符串”是字母异位词集合的**唯一规范化表示（Canonical Form）**。

1. **充分性**：若字符串 $S_1$ 和 $S_2$ 是字母异位词，根据定义，它们包含的字符种类和每个字符出现的次数完全相同。由于排序算法是确定的（Deterministic），相同的字符集合排序后必然得到相同的序列。因此，$sort(S_1) = sort(S_2)$。
2. **必要性**：若 $sort(S_1) = sort(S_2)$，说明 $S_1$ 和 $S_2$ 排序后的字符序列完全一致。这意味着 $S_1$ 只是 $S_2$ 的一个排列（Permutation），符合字母异位词的定义。
3. **结论**：通过哈希表将具有相同规范化表示（排序后字符串）的元素归为一类，可以不重不漏地完成分组。

---

## 4. 编程中需要注意的小细节 (易错点)

1. **Python 中 `sorted()` 的返回值**：
   `sorted("eat")` 返回的是一个列表 `['a', 'e', 't']`。在 Python 中，**列表是不可哈希的（unhashable）**，不能直接作为字典的 `key`。必须使用 `"".join(sorted(s))` 将其转换回字符串，或者转换为元组 `tuple(sorted(s))`。

2. **哈希表的初始化**：
   使用 `result = {}` 时，需要判断 `if key not in result`。为了代码更简洁，可以使用 `collections.defaultdict(list)`，这样可以直接执行 `result[key].append(s)` 而无需判断。

3. **空字符串处理**：
   输入 `[""]` 是合法情况。排序后依然是 `""`，哈希表会存为 `{"": [""]}`，最终返回 `[[""]]`，逻辑完全正确。

4. **性能优化思考**：
   如果字符串非常长，排序 $O(K \log K)$ 会变慢。此时可以改用**计数法**：统计每个字母出现的次数（长度为 26 的数组），将该数组转为元组作为 `key`。计数法的时间复杂度为 $O(N \cdot K)$。
