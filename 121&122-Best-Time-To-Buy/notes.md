# LeetCode 121 & 122: 买卖股票的最佳时机 I & II

## 1. 题目描述

**LeetCode 121 (只能交易一次):**
给定一个数组 `prices` ，它的第 `i` 个元素 `prices[i]` 表示一支给定股票第 `i` 天的价格。
你只能选择 **某一天** 买入这只股票，并选择在 **未来的某一个不同的日子** 卖出该股票。返回你可以从这笔交易中获取的最大利润。

**LeetCode 122 (可以交易无数次):**
给你一个整数数组 `prices` 。在每一天，你可以决定是否购买和/或出售股票。你在任何时候 **最多** 只能持有 **一股** 股票。你也可以先购买，然后在 **同一天** 出售。返回你能获得的 **最大** 利润。

---

## 2. 算法基本原理与多解法对比

这两道题都存在两种主流解法：**动态规划 (通用解法)** 和 **贪心/双指针 (特化极速解法)**。

### 对于 121 题 (交易 1 次)

* **解法 A：单向遍历 / 贪心**
  * **原理**：维护一个历史最低价格 `min_price`，每天都假设今天卖出，计算利润 `price - min_price`，并更新全局最大利润 `max_profit`。
  * **复杂度**：时间 $O(N)$，空间 $O(1)$。
* **解法 B：动态规划**
  * **原理**：`dp[i][0]` 表示不持股，`dp[i][1]` 表示持股。因为只能买一次，所以买入时的状态转移为 `dp[i][1] = max(dp[i-1][1], -prices[i])`（注意这里是 `0 - prices[i]`，因为买入前利润必为 0）。

### 对于 122 题 (交易无数次)

* **解法 A：动态规划**
  * **原理**：买入时的状态转移为 `dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i])`（因为可以多次买卖，买入前可能已经有利润了）。
  * **复杂度**：时间 $O(N)$，空间 $O(N)$（可优化至 $O(1)$）。
* **解法 B：贪心算法**
  * **原理**：既然可以无限次交易，那么只要明天的价格比今天高，我就今天买、明天卖。把所有“上坡”的利润全部收集起来，就是最大利润。
  * **复杂度**：时间 $O(N)$，空间 $O(1)$。

### 算法优劣对比

| 算法       | 优势                               | 劣势                             | 性能表现            |
|:-------- |:-------------------------------- |:------------------------------ |:--------------- |
| **动态规划** | 思想通用，一套模板可以解决所有股票变种题（含冷冻期、手续费等）。 | 代码稍长，如果不做状态压缩，空间复杂度较高 $O(N)$。  | 稍慢（因为有状态转移的开销）。 |
| **贪心算法** | 代码极简，运行速度极快，空间复杂度严格 $O(1)$。      | 思想特化，一旦题目增加限制（如限制交易2次），贪心立刻失效。 | 极快（击败 90%+）。    |

## 3. 数学归纳法与算法证明

这里重点证明 **122题 贪心算法** 的正确性。

**命题**：对于任意数组 `prices`，收集所有相邻两天的正差值 $\sum \max(0, prices[i] - prices[i-1])$，等于全局最大利润。

**证明 (利用差分数组的裂项相消)**：
假设在第 $i$ 天买入，第 $j$ 天卖出 ($j > i$) 是一次最优交易，其利润为 $P_j - P_i$。
根据数学等式，我们可以将这段利润展开为每天的差值之和：
$P_j - P_i = (P_{i+1} - P_i) + (P_{i+2} - P_{i+1}) + \dots + (P_j - P_{j-1})$

* 如果这段区间是单调递增的，那么每天的差值都是正数，贪心算法会把它们全部收集，总和正好等于 $P_j - P_i$。
* 如果这段区间中间有下跌（即存在 $P_k - P_{k-1} < 0$），贪心算法会**跳过**这个负数（相当于在下跌前卖出，下跌后重新买入）。这样收集到的利润总和将 **严格大于** 一直持股不动的利润 $P_j - P_i$。
* **结论**：贪心算法等价于在每一个局部最低点买入，局部最高点卖出，因此必然能得到全局最大利润。

## 4. 编程中需要注意的小细节 (易错点回顾)

1. **DP 数组的初始化陷阱**：
   在 Python 中初始化二维数组时，切忌使用 `dp = [prices, prices]` 或 `dp = [[0]*2] * n`。这会导致列表引用同一块内存地址，或者直接修改原输入数据。
   标准的二维数组初始化应为：`dp = [[0, 0] for _ in range(n)]`。
2. **DP 的空间优化**：
   如果开辟长度为 $N$ 的 `dp` 数组，空间复杂度为 $O(N)$，内存消耗较大。
   观察状态转移方程：`dp[n+1]` 只依赖于 `dp[n]`。因此，我们只需要两个变量 `cash` (不持股) 和 `hold` (持股) 滚动更新即可，将空间复杂度从 $O(N)$ 降维到 $O(1)$。
3. **121 题的初始值设定**：
   在 121 题中，将最低价格初始化为 `float("inf")`，最大利润初始化为 `0` 是非常标准的做法，能够完美处理数组第一天的边界情况，避免引入额外的 `if` 判断。
