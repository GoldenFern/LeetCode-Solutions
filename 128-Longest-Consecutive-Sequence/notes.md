# LeetCode 128: 最长连续序列 (Longest Consecutive Sequence)

## 1. 题目描述

给定一个未排序的整数数组 `nums` ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 `O(n)` 的算法解决此问题。

**示例 1:**
> 输入: nums = [100,4,200,1,3,2]
> 输出: 4
> 解释: 最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。

**示例 2:**
> 输入: nums = [0,3,7,2,5,8,4,6,0,1]
> 输出: 9

**示例 3:**
> 输入: nums = [1,0,1,2]
> 输出: 3

**提示：**
* $0 \le nums.length \le 10^5$
* $-10^9 \le nums[i] \le 10^9$

---

## 2. 算法基本原理 (四种解法分析)

本题虽然要求 $O(n)$ 的时间复杂度，但为了拓宽思路，这里列出四种常见的解法及其复杂度分析：

### 解法一：哈希表 (最优解，符合 $O(n)$ 要求)
* **核心思想**：将数组元素存入哈希集合（Set）中去重并实现 $O(1)$ 的查找。遍历集合，**只从每个连续序列的起点开始向后枚举**。
* **时间复杂度**：$O(n)$。虽然有一个嵌套的 `while` 循环，但因为我们只对序列的“起点”进入 `while` 循环，数组中的每个数最多只会被访问两次（一次是外层 `for` 循环，一次是内层 `while` 循环），因此均摊时间复杂度为 $O(n)$。
* **空间复杂度**：$O(n)$，用于存储哈希集合。

### 解法二：动态规划 (基于排序)
* **核心思想**：先将数组排序，然后定义 `dp[i]` 表示以 `nums[i]` 结尾的连续序列的长度。遍历排序后的数组：如果当前元素比前一个元素大 1，则 `dp[i] = dp[i-1] + 1`；如果相等，则继承前一个的长度 `dp[i] = dp[i-1]`；否则断开连续，重置为 `dp[i] = 1`。
* **时间复杂度**：$O(n \log n)$。瓶颈在于 `nums.sort()` 的排序操作，后续的 DP 状态转移只需遍历一次数组 $O(n)$。
* **空间复杂度**：$O(n)$，需要一个长度为 $n$ 的 `dp` 数组来记录状态（另外排序本身也可能需要 $O(\log n)$ 到 $O(n)$ 的栈空间）。

### 解法三：滑动窗口 / 双指针 (基于排序，空间优化)
* **核心思想**：与解法二的动态规划思路类似，先将数组排序。但不使用 $O(n)$ 的 `dp` 数组，而是仅用一个变量 `current_streak` 记录当前窗口/连续段的长度。遇到连续元素则加 1，遇到重复元素跳过，遇到断开的元素则结算并重置为 1。
* **时间复杂度**：$O(n \log n)$。排序的复杂度为 $O(n \log n)$，后续遍历为 $O(n)$。
* **空间复杂度**：$O(1)$ 或 $O(n)$，取决于排序算法的底层实现。相比解法二，省去了 `dp` 数组的开销。

### 解法四：暴力枚举
* **核心思想**：遍历数组中的每一个数 `x`，然后不断在数组中暴力搜索 `x+1`, `x+2`, ... 是否存在。
* **时间复杂度**：$O(n^3)$。外层遍历 $O(n)$，内层匹配连续序列最坏 $O(n)$，每次在数组中线性查找下一个数 $O(n)$。
* **空间复杂度**：$O(1)$。

## 3. 编程中需要注意的小细节 (易错点)

1. **必须排除不是起点的元素 (核心剪枝)**：
   在哈希表解法中，`if num - 1 not in num_set:` 这一句是算法的灵魂。
   * 如果不加这个判断，遇到 `[1, 2, 3, 4]` 时，遍历到 `2` 会往后找 `3, 4`，遍历到 `3` 会往后找 `4`，导致时间复杂度退化为 $O(n^2)$。
   * 加上这个判断后，只有当 `num` 是一个序列的**绝对起点**（即它前面的那个数不存在于集合中）时，才开始计算。这保证了内层 `while` 循环总共执行的次数不超过 $n$。

2. **深刻理解 `set` 中的 `in` 操作**：
   在 Python 中，`set` 的底层实现是哈希表。
   * 使用 `num in num_set` 的时间复杂度是 **$O(1)$**，这是符合哈希表特性的。
   * 如果你不把 `list` 转换为 `set`，直接写 `num in nums`（在列表中查找），时间复杂度是 **$O(n)$**，这会导致整体算法退化为 $O(n^2)$。因此，**第一步 `num_set = set(nums)` 是绝对不能省略的**。

3. **空数组特判**：
   如果输入 `nums = []`，直接返回 `0`。虽然在 Python 中 `for num in num_set:` 遇到空集合会直接跳过并返回初始化的 `0`，但明确边界条件是一个好习惯。
