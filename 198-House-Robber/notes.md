# LeetCode 198: 打家劫舍 (House Robber)

## 1. 题目描述

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是**相邻的房屋装有相互连通的防盗系统**。如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。

给定一个代表每个房屋存放金额的非负整数数组，计算你 **不触动警报装置的情况下** ，一夜之内能够偷窃到的最高金额。

---

## 2. 算法基本原理：动态规划 (Dynamic Programming)

这是一个典型的**最优子结构**问题，可以通过动态规划将大问题分解为小问题。

* **状态定义**：设 $f(n)$ 为偷到第 $n$ 间房时能获得的最大金额。
* **状态转移方程**：对于第 $n$ 间房，我们有两种选择：
  1. **不偷第 $n$ 间房**：那么最大金额等于偷到前一间房的最大金额，即 $f(n) = f(n-1)$。
  2. **偷第 $n$ 间房**：那么我们不能偷第 $n-1$ 间房，最大金额等于前两间房的最大金额加上当前房间金额，即 $f(n) = f(n-2) + \text{nums}[n]$。
  * 最终决策：$f(n) = \max(f(n-1), f(n-2) + \text{nums}[n])$。
* **空间优化**：由于 $f(n)$ 只依赖于 $f(n-1)$ 和 $f(n-2)$，我们不需要存储整个数组，只需两个变量来记录前两个状态即可，将空间复杂度从 $O(n)$ 降至 $O(1)$。

---

## 3. 编程中需要注意的小细节 (易错点)

1. **变量更新的顺序（致命错误）**：
   在更新状态变量时，必须注意先后顺序。
   
   * **错误写法**：
     
     ```python
     f_n_minus_1 = f_n
     f_n_minus_2 = f_n_minus_1 # 此时 f_n_minus_1 已经是新值了，导致数据覆盖
     ```
   * **正确写法**：
     使用临时变量保存旧值，或者利用 Python 的元组解包特性：
     
     ```python
     f_n_minus_2, f_n_minus_1 = f_n_minus_1, f_n
     ```

2. **边界条件处理**：
   
   * 如果只有 1 间房，直接返回 `nums[0]`。
   * 如果只有 2 间房，返回 `max(nums[0], nums[1])`。
   * 初始化时，$f(n-2)$ 对应第 1 间房，$f(n-1)$ 对应前 2 间房的最大值。

3. **循环起点**：
   如果手动处理了前两个边界，循环应从 `nums[2:]` 开始。
