# LeetCode 209: 长度最小的子数组 (Minimum Size Subarray Sum)

## 1. 题目描述

给定一个含有 `n` 个正整数的数组和一个正整数 `target` 。

找出该数组中满足其总和大于等于 `target` 的长度最小的 **子数组** `[numsl, numsl+1, ..., numsr-1, numsr]` ，并返回其长度。如果不存在符合条件的子数组，返回 `0` 。

**示例 1:**
> 输入: target = 7, nums = [2,3,1,2,4,3]
> 输出: 2
> 解释: 子数组 [4,3] 是该条件下的长度最小的子数组。

**示例 2:**
> 输入: target = 4, nums = [1,4,4]
> 输出: 1

**示例 3:**
> 输入: target = 11, nums = [1,1,1,1,1,1,1,1]
> 输出: 0

**提示：**
* $1 \le target \le 10^9$
* $1 \le nums.length \le 10^5$
* $1 \le nums[i] \le 10^4$

---

## 2. 错误解法与反例分析

**初始思路（错误）**：
试图使用相向双指针（碰撞指针），初始化 `left = 0, right = len(nums) - 1`，计算整个数组的和。如果和大于目标值，则比较 `nums[left]` 和 `nums[right]` 的大小，舍弃较小的那一个，向中间收缩。

**致命反例**：
`target = 7, nums = [5, 1, 3, 4, 2, 1, 1]`
*   **最优解**应该是 `[3, 4]`，长度为 2。
*   **错误算法轨迹**：初始和为 17，满足条件。比较两端，`5 > 1`，所以右边界左移。右边界会一直因为右侧的值较小而不断向左收缩（6 -> 5 -> 4 -> ... -> 1），最终错过了包含中间部分 `[3, 4]` 的最优解，给出了错误的长度 3（子数组 `[5, 1, 3]`）。

**根本原因**：
寻找“连续子数组”的和，两端元素本身的大小对比（例如 `5 > 1`）并不能为“如何缩小区间才能保持和 $\ge target$ 且长度最短”提供任何数学上的单调性指导。这种贪心策略是无效的。

---

## 3. 算法基本原理：滑动窗口 (Sliding Window)

滑动窗口是处理**连续子串/子数组**问题的经典优化策略，本质上是同向双指针。

*   **核心思想**：维护一个窗口 `[left, right]`，`right` 主动扩张搜索范围，`left` 被动收缩优化解。
*   **扩张逻辑**：`right` 指针不断向右移动，把元素加入窗口累加，直到当前窗口的和 `current_sum >= target`。
*   **收缩逻辑**：一旦条件满足，记录当前长度，并尝试将 `left` 指针向右移动以缩小窗口。如果缩小后依然满足条件，更新 `min_len`。重复此过程直到窗口和小于 `target`。
*   **时间复杂度**：$O(n)$。每个元素最多进出窗口各一次。

---

## 4. 数学证明与逻辑严密性分析

**为什么滑动窗口不会漏掉最优解？**

设 $S(i, j)$ 为数组从索引 $i$ 到 $j$ 的元素总和（$i \le j$）。
假设在某步，我们固定左边界为 $L$，右指针向右扫描，第一次在位置 $R$ 满足 $S(L, R) \ge target$。

1.  此时，区间 $[L, R]$ 是以 $L$ 为左端点的**最短**合法子数组。
2.  对于任何 $R' > R$，$S(L, R')$ 必然也 $\ge target$（因为数组全是正整数），但其长度 $R' - L + 1$ 必然大于 $R - L + 1$。
3.  因此，在固定 $L$ 的情况下，我们无需再检查任何比 $R$ 更靠右的端点。
4.  为了寻找更短的可能，我们**必须**增大 $L$（即 `left++`），去探索以 $L+1, L+2 \dots$ 为起点的情况。
通过这种方式，算法排除了所有冗余的计算，确保在 $O(n)$ 时间内覆盖了所有可能的潜在最优解。

---

## 5. 核心总结：滑动窗口 vs 碰撞指针

在编程实现中，如何选择双指针的初始化方向取决于解空间的结构：

1.  **滑动窗口 (`left = 0, right = 0`)**：
    *   **适用场景**：寻找满足条件的**连续子段**（子数组/子串）。
    *   **逻辑**：像一个滑动的探测器，通过 `right` 增加新信息，`left` 移除旧信息，维护一段动态的连续区间。

2.  **碰撞指针 (`left = 0, right = n - 1`)**：
    *   **适用场景**：寻找**两个元素的组合**，通常不要求连续。
    *   **逻辑**：利用题目中的**单调性**（如排序数组）或**边界制约**（如木桶效应）。通过移动指针来“剪枝”，即排除掉那些在数学上绝对不可能成为最优解的边界组合。
