# LeetCode 279: 完全平方数 (Perfect Squares)

## 1. 题目描述

给你一个整数 `n` ，返回 **和为 `n` 的完全平方数的最少数量** 。

**完全平方数** 是一个整数，其值等于另一个整数的平方；换句话说，其值等于一个整数乘以自己。例如，1、4、9 和 16 都是完全平方数，而 3 和 11 不是。

**示例 1:**

> 输入：n = 12
> 输出：3 
> 解释：12 = 4 + 4 + 4

**示例 2:**

> 输入：n = 13
> 输出：2
> 解释：13 = 4 + 9

**提示：**

* $1 \le n \le 10^4$

---

## 2. 算法基本原理

本题有两种经典解法：动态规划（通用解法）和数学定理（极速解法）。

### 方法一：动态规划 (Dynamic Programming)

* **核心思想**：这是一个典型的完全背包问题变种。要求组成数字 `i` 的最少平方数数量，可以枚举最后一个加上的平方数 $j^2$，那么状态转移方程为：`dp[i] = min(dp[i], dp[i - j*j] + 1)`。
* **复杂度**：时间复杂度为 $O(n\sqrt{n})$，空间复杂度为 $O(n)$。

### 方法二：数学定理 (Math)

利用数论中的两个著名定理，我们可以直接将答案锁定在 1、2、3、4 之中。

1. **拉格朗日四平方和定理 (Lagrange's four-square theorem)**：
   任何一个正整数都可以表示为至多四个正整数的平方和。这意味着本题的答案**最大只能是 4**。
2. **勒让德三平方和定理 (Legendre's three-square theorem)**：
   当且仅当 $n \neq 4^a(8b+7)$ 时，正整数 $n$ 可以表示为至多三个正整数的平方和。换言之，如果 $n = 4^a(8b+7)$，那么它绝对不能用 1 个、2 个或 3 个平方数表示，**答案必定是 4**。

**数学法逻辑分支**：

1. 如果 $n$ 本身是完全平方数 $\rightarrow$ 答案为 **1**。
2. 如果 $n$ 满足 $n = 4^a(8b+7)$ $\rightarrow$ 答案为 **4**。
3. 枚举 $i \in [1, \sqrt{n}]$，如果 $n - i^2$ 是完全平方数 $\rightarrow$ 答案为 **2**。
4. 排除以上所有情况 $\rightarrow$ 答案为 **3**。
* **复杂度**：时间复杂度降至 $O(\sqrt{n})$，空间复杂度为 $O(1)$。

## 3. 数学归纳法与算法证明

### 动态规划的最优子结构证明

**命题**：`dp[i]` 记录的是组成数字 `i` 的最少完全平方数个数。

* **基础步骤**：`dp[0] = 0`，0 个平方数组成 0，显然成立。
* **归纳假设**：假设对于所有 $k < i$，`dp[k]` 都已经是最优解。
* **归纳步骤**：对于数字 $i$，它必然是由某个完全平方数 $j^2$ 和剩余部分 $i - j^2$ 组成。因为 $i - j^2 < i$，根据归纳假设，`dp[i - j^2]` 已经是组成剩余部分的最优解。我们遍历所有满足 $j^2 \le i$ 的 $j$，取 `dp[i - j^2] + 1` 的最小值，必然能覆盖所有组合情况并得到 `dp[i]` 的全局最优解。

## 4. 编程中需要注意的小细节 (我的踩坑记录)

回顾我最初的尝试，在推导 DP 时犯了几个非常典型的思维误区，记录如下：

1. **DP 状态存储的数据结构选择 (Dict vs Array)**：
   
   * **我的错误直觉**：第一次尝试中，我使用了字典 `minimize_result = {1:1}` 来记录状态，并且试图用 `while` 循环只把完全平方数（4, 9, 16...）加入字典并赋值为 1。这导致我不知道该如何填补非完全平方数（如 2, 3, 5）的空白。
   * **正确逻辑**：对于这种连续整数的状态转移，**一维数组 (List)** 是最佳选择。建立 `dp` 数组并将 `dp[0] = 0` 作为基石。这样任何数字 $i$ 都可以通过 `dp[i - square]` 顺藤摸瓜找到前面的状态。

2. **循环结构的缺失 (如何填满 DP 数组)**：
   
   * **我的错误直觉**：第一次代码写到 `while k_square < n:` 就写不下去了，因为单层循环只能处理完全平方数本身。
   * **正确逻辑**：必须使用**双层循环**。外层循环 `for i in range(1, n + 1)` 负责遍历每一个需要求解的数字（背包容量）；内层循环 `for square in squares` 负责尝试放入不同的完全平方数（物品）。注意索引与数字的区别。
