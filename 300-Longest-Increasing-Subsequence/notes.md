# LeetCode 300: 最长递增子序列 (Longest Increasing Subsequence)

## 1. 题目描述

给你一个整数数组 `nums` ，找到其中最长严格递增子序列的长度。

**子序列** 是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如，`[3,6,2,7]` 是数组 `[0,3,1,6,2,2,7]` 的子序列。

**示例 1：**
> 输入：nums = [10,9,2,5,3,7,101,18]
> 输出：4
> 解释：最长递增子序列是 [2,3,7,101]，因此长度为 4 。

**示例 2：**
> 输入：nums = [0,1,0,3,2,3]
> 输出：4

**提示：**
* $1 \le nums.length \le 2500$
* $-10^4 \le nums[i] \le 10^4$

---

## 2. 算法基本原理：贪心 + 二分查找 (Greedy + Binary Search)

本题的基础解法是 $O(n^2)$ 的动态规划，但存在更优的 **$O(n \log n)$** 解法。

* **核心数据结构**：维护一个数组 `tails`。`tails[i]` 的定义是：**在所有长度为 `i+1` 的递增子序列中，末尾元素的最小值**。
* **贪心策略（潜力优化）**：为了让递增子序列尽可能长，我们希望子序列的末尾数字尽可能小，这样后续才更有可能接上新的数字。
* **操作规则（大的接后面，小的换中间）**：
  遍历原数组 `nums` 中的每个元素 `x`：
  1. 如果 `x` 比 `tails` 中所有的数都大，说明找到了一个更长的递增子序列，直接将 `x` 追加到 `tails` 末尾。
  2. 如果 `x` 不大于 `tails` 的最大值，则在 `tails` 中找到**第一个大于或等于 `x` 的元素**，并用 `x` 替换它。
* **时间复杂度**：遍历数组需要 $O(n)$，每次在 `tails` 中寻找替换位置使用二分查找需要 $O(\log n)$，总时间复杂度为 $O(n \log n)$。空间复杂度为 $O(n)$。

---

## 3. 算法证明：为什么 `tails` 始终有序且能使用二分查找？

二分查找的前提是数组必须有序。我们需要证明：**在任何时刻，`tails` 数组都是严格单调递增的。**

**数学归纳法证明：**

* **基础步骤 (Base Case)**：
  当处理第一个元素时，`tails` 只有一个元素，显然是严格递增的。

* **归纳假设 (Inductive Step)**：
  假设在处理第 $k$ 个元素之前，`tails` 数组是严格递增的。现在处理第 $k$ 个元素 $x$：
  
  **情况 A：$x > tails[-1]$**
  我们将 $x$ 追加到 `tails` 末尾。因为 $x$ 严格大于原数组的最大值，追加后 `tails` 依然保持严格递增。
  
  **情况 B：$x \le tails[-1]$**
  我们需要找到第一个满足 $tails[i] \ge x$ 的元素，并将其替换为 $x$。
  因为 $tails[i]$ 是**第一个**大于等于 $x$ 的数，所以它前面的数必然严格小于 $x$（即 $tails[i-1] < x$）。
  替换前：$tails[i-1] < tails[i] < tails[i+1]$
  替换后：$tails[i-1] < x \le tails[i] < tails[i+1]$
  由于我们是用更小的 $x$ 替换了 $tails[i]$，且 $x$ 依然严格大于 $tails[i-1]$，因此替换操作**不会破坏数组的严格递增性**。

* **结论**：
  无论执行追加还是替换，`tails` 始终保持严格递增。因此，我们可以在 `tails` 上安全地使用二分查找。最终 `tails` 的长度即为最长递增子序列的长度。

## 4. 编程中需要注意的小细节 (易错点)

1. **`tails` 数组存的不是 LIS 本身**：
   这是最容易产生的误解。例如输入 `nums = [4, 5, 6, 3]`，最终 `tails = [3, 5, 6]`。虽然长度 3 是正确的，但 `[3, 5, 6]` 并不是原数组的有效子序列。`tails` 只是用来记录“各个长度下最优的末尾值”。
2. **二分查找的边界条件**：
   我们需要实现的是“寻找左侧边界”的二分查找（即找到第一个 $\ge x$ 的位置）。
   * 循环条件通常写为 `while left < right:`。
   * 当 `tails[mid] < x` 时，说明目标在右侧，`left = mid + 1`。
   * 当 `tails[mid] >= x` 时，说明目标在左侧或就是 `mid`，因此 `right = mid`（注意这里不能是 `mid - 1`，否则会漏掉正确答案）。
3. **Python 内置库**：
   在实际工程或笔试中，可以直接使用 Python 的 `bisect` 库中的 `bisect_left(tails, x)` 来代替手写二分查找，代码会更简洁。
