# LeetCode 438: 找到字符串中所有字母异位词 (Find All Anagrams in a String)

## 1. 题目描述

给定两个字符串 `s` 和 `p`，找到 `s` 中所有 `p` 的 **异位词** 的子串，返回这些子串的起始索引。不考虑答案输出的顺序。

**异位词** 指由相同字母重排列形成的字符串（包括相同的字符串）。

**示例 1:**

> 输入: s = "cbaebabacd", p = "abc"
> 输出: [0,6]
> 解释:
> 起始索引等于 0 的子串是 "cba", 它是 "abc" 的异位词。
> 起始索引等于 6 的子串是 "bac", 它是 "abc" 的异位词。

**示例 2:**

> 输入: s = "abab", p = "ab"
> 输出: [0,1,2]
> 解释:
> 起始索引等于 0 的子串是 "ab", 它是 "ab" 的异位词。
> 起始索引等于 1 的子串是 "ba", 它是 "ab" 的异位词。
> 起始索引等于 2 的子串是 "ab", 它是 "ab" 的异位词。

**提示:**
* $1 \le s.length, p.length \le 3 \times 10^4$
* `s` 和 `p` 仅包含小写字母

---

## 2. 算法基本原理：定长滑动窗口 + 哈希表

* **核心思想**：异位词的本质是**字符种类和数量完全相同**，与字符顺序无关。因此，我们可以维护一个长度固定为 `len(p)` 的滑动窗口，统计窗口内的字符频率，并与 `p` 的字符频率进行比较。
* **时间复杂度优化**：
  * **错误思路复盘**：如果每次都将 `p` 转换为列表并使用 `list.remove()` 来匹配字符，由于 `remove()` 本身是 $O(K)$ 的操作，嵌套在循环中会导致 $O(N \times K)$ 的复杂度，必然超时。
  * **正确思路**：使用哈希表（`collections.Counter`）记录频率。窗口每次向右滑动一格，只需执行两次 $O(1)$ 的操作（右边界字符频率 $+1$，左边界字符频率 $-1$）。整体时间复杂度降为 $O(N)$。

---

## 3. 数学归纳法对算法的证明

**命题 $P(k)$**：在第 $k$ 次循环结束时，`window_count` 准确记录了字符串 `s` 中区间 $[k - len(p) + 1, k]$（即长度为 $len(p)$ 的子串）的字符频率。

* **基础步骤 (Base Case)**：
  在进入主循环前，代码预先将前 $len(p) - 1$ 个字符加入了 `window_count`。此时窗口尚未达到目标长度，为后续滑动做好了准备。

* **归纳假设 (Inductive Step)**：
  假设在第 $k-1$ 步，窗口正确记录了前一个状态的频率。
  在第 $k$ 步（主循环中）：
  1. **右边界进入**：`s[k]` 进入窗口，`window_count[s[k]] += 1`。此时窗口包含了 $len(p)$ 个字符，正好对应当前需要判断的子串。
  2. **状态判定**：此时执行 `window_count == p_count`，准确无误地判断了当前子串是否为异位词。
  3. **左边界移出**：为了给下一步做准备，必须将最左侧的字符 `s[k - len(p) + 1]` 移出窗口，即 `window_count[left_char] -= 1`。
  
* **结论**：
  通过数学归纳法可知，这种“进一个、判一次、出一个”的交替操作，保证了每次执行 `==` 比较时，`window_count` 都精确代表了一个长度为 $len(p)$ 的连续子串，且没有遗漏任何一个可能的起始位置。

## 4. 编程中需要注意的小细节 (易错点)

1. **极其隐蔽的 Python 版本特性 (`Counter` 的比较)**：
   * 在代码中，当左边界字符移出窗口时，执行了 `window_count[left_char] -= 1`。如果某个字符的频率减到了 `0`，这个键（Key）**依然存在**于字典中（例如 `{'a': 1, 'b': 0}`）。
   * **在 Python 3.10 之前**：`Counter({'a': 1, 'b': 0}) == Counter({'a': 1})` 会返回 `False`！因此在旧版本中，必须加上 `if window_count[left_char] == 0: del window_count[left_char]` 才能通过测试。
   * **在 Python 3.10 及以后**：官方修改了 `Counter` 的底层逻辑，**在进行 `==` 比较时，会自动忽略值为 `0` 的键**。LeetCode 目前使用的是 Python 3.10+ 环境，所以你的代码即使没有 `del` 也能完美 AC！这是一个非常值得炫耀的语言细节。

2. **边界条件拦截**：
   如果 `len(s) < len(p)`，`s` 中绝对不可能存在 `p` 的异位词。必须在开头加上 `if s_len < p_len: return []`，否则后续的窗口初始化和索引计算会发生越界错误。

3. **替代方案（数组哈希）**：
   由于题目明确指出只包含小写字母，在 C++ 或 Java 中，通常会使用两个长度为 26 的数组 `int[26]` 来代替哈希表，通过比较数组是否相等来判断异位词，这样运行速度会比哈希表更快。
